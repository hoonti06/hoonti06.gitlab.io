- Spring data JPA
	- JPA는 Java Persistence API로 ORM 방식의 Java 인터페이스이다
		- 구현체는 Hibernate이다.
			- Hibernate 개발자가 JPA도 같이 개발했기 때문에 Hibernate가 JPA의 많은 부분을 구현하고 있다.
		- SQL과 데이터 중심에서 객체 중심으로 변한다
		- JPA는 PK에 대해서는 method가 제공되지만, 그 외에는 JPQL(Java Persistence Query Language)을 사용해야 한다
		```java
		// user는 entity 이름이고, select 대상은 alias인 u 자체가 된다.
		String jpql = "select u from user u"; 
		```
	- Spring data JPA는 Spring에서 제공하는 JPA를 한 번 더 감싼 wrapper이다.
		- JPA를 편리하게 사용할 수 있도록 도와준다
	- Spring data JPA의 경우 복잡한 query가 아닌 경우에는 method 이름의 형식을 통해 query문을 대신할 수 있다.
	- **JPA를 먼저 학습하고나서 Spring data JPA를 학습해야 한다**
	- 복잡한 동적 query는 Querydsl이라는 라이브러리를 사용하면 된다.
	- 위 조합으로도 해결하기 어려운 query는 JPArk wprhdgksms 네이티브 query 또는 JdbcTemplate을 사용하면 된다.
